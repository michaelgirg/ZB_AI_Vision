# Quantized Inference

The ARM deployment path uses post-training quantization of the PyTorch MLP.
This gives the project a real PyTorch-to-embedded-AI story instead of relying
only on float inference.

## Data Format

The FPGA threshold output is converted into binary classifier inputs:

```text
0x00 -> 0
0xFF -> 1
```

The first layer consumes those integer values directly.

The upgraded threshold+Sobel path uses 1568 normalized features:

```text
threshold image: 784 bytes, values 0 or 255
Sobel image:     784 bytes, values 0..255
combined input:  threshold bytes followed by Sobel bytes
```

For that model, the integer input uses `0..255` with a scale of `1/255`.

## Integer Pipeline

```text
input uint8 0/1
  -> fc1 int8 weights + int32 bias
  -> int32 accumulator
  -> ReLU
  -> integer multiplier/shift requantization to hidden activation 0..127
  -> fc2 int8 weights + int32 bias
  -> int32 logits
  -> argmax prediction
```

The final integer logits can be used directly for prediction. The exported
floating-point scales are for debug and Python/C comparison.

## Current Baseline

Generated by:

```powershell
.\.venv\Scripts\python pytorch\export_quantized.py --checkpoint generated\model\threshold_mlp.pt
.\.venv\Scripts\python scripts\validate_quantized.py
```

Current result:

```text
fixed-point test accuracy: 94.68%
float/fixed prediction agreement: 99.84%
max absolute dequantized logit error: 0.3057
```

Generated files:

```text
generated/headers/model_weights_quantized.h
generated/headers/model_quantized_golden.h
generated/model/quantized_metrics.json
```

## Threshold + Sobel Model

Generated by:

```powershell
.\.venv\Scripts\python pytorch\train_model.py --feature-mode threshold_sobel --hidden 96 --epochs 3
.\.venv\Scripts\python pytorch\export_quantized.py --checkpoint generated\model\threshold_sobel_mlp.pt --header generated\headers\model_weights_threshold_sobel_quantized.h --golden-header generated\headers\model_threshold_sobel_quantized_golden.h --metadata generated\model\threshold_sobel_quantized_metrics.json
.\.venv\Scripts\python scripts\validate_quantized.py --metadata generated\model\threshold_sobel_quantized_metrics.json --min-fixed-accuracy 0.95 --min-agreement 0.99
```

Current result:

```text
float test accuracy: 95.94%
fixed-point test accuracy: 95.98%
float/fixed prediction agreement: 99.86%
max absolute dequantized logit error: 0.2210
```

Generated files:

```text
generated/headers/model_weights_threshold_sobel_quantized.h
generated/headers/model_threshold_sobel_quantized_golden.h
generated/model/threshold_sobel_quantized_metrics.json
```

The golden headers store exported sample labels, predictions, and integer
logits so the Vitis app can compare ARM C inference against the Python
fixed-point reference.

The hidden-layer requantization is exported as integer constants:

```text
MODEL_HIDDEN_REQUANT_MULTIPLIER
MODEL_HIDDEN_REQUANT_SHIFT
```

The ARM implementation should use `int64_t` for the multiply before shifting.

The C implementation lives in:

```text
vitis/classifier.c
vitis/classifier.h
```

The host/Vitis validation harness is:

```text
vitis/test_classifier_host.c
```
